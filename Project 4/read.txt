Project #4 (25 pts): Build a Technical Documentation Page
Purpose
You will complete the freeCodeCamp certification project Build a Technical Documentation Page, passing all tests, add additional class requirements,  then publish it as a real webpage in W3Schools Spaces and submit the public URL in Blackboard.

This project should feel like a real, helpful rulebook someone could use to learn a game.

Topic (Required)
Create a technical documentation page that explains how to play a board game.

Choose ONE game
Pick a board game you know well or want to teach to someone else.

Required personal connection (top of page)
At the top of your documentation page, include a short “Why this game?” intro (5–8 sentences) that answers:

Why did you choose this game?
How long have you been playing it (or how did you discover it)?
Who would you recommend it to (and why)?
Required Documentation Sections (Must Include)
Your page must include sections that clearly explain the game. At minimum, include:

Overview (what the game is and the goal)
What You Need (components / materials)
Setup (step-by-step)
How to Play (turn structure / rules)
Scoring / Winning (how the game ends and how you win)
Common Mistakes / Tips (at least 3 practical tips)
FAQ / Edge Cases (at least 3 questions with answers)
These should be written in your own words. Don’t paste full rules from a website.

Navigation Expectations (Documentation Style)
Your page should function like real documentation:

A clear navigation menu
Sections are easy to scan
Headings are meaningful and consistent
Visual Expectations (Quality Matters)
To earn a strong score:

Use clean formatting and consistent spacing
Keep paragraphs readable (not giant walls of text)
Use lists where appropriate
Make the page look intentional and “finished”
What to Complete (freeCodeCamp)
Complete Build a Technical Documentation Page
Ensure all freeCodeCamp tests pass
Publish on W3Schools Spaces (Required)
Folder name (must be exact)
project4-technical-docs
Required files (must be exact)
index.html
styles.css (required)
Required CSS link in HTML
 
<link rel="stylesheet" href="styles.css">
 
What to Submit (Blackboard)
Submit one public URL that works in an incognito/private window:

https://your-name.w3spaces.com/project4-technical-docs/

🚫 Do not submit preview/editor links.
